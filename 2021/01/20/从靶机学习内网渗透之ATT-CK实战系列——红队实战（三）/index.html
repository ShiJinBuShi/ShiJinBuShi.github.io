    前言

    之前在看内网渗透的知识一直也没有机会实践，刚好学校课程设计里面有个内网渗透与加固方案的课题，于是找了个红日的靶场准备搞一下，顺便写篇文件记录总结。小白初识内网，大佬们多多指教。

    一、环境搭建

    靶机介绍与下载地址：http://vulnstack.qiyuanxuetang.net/vuln/detail/5/

    打开虚拟机镜像为挂起状态，CentOS为出网机，目标是域控内的一份重要文件。按下图配好网络环境后运行五台机器即可。

    渗透选择使用Kali，桥接到物理网卡。

    三、边界突破

    1、扫扫扫

    我这里直接使用joomscan，GitHub地址：https://github.com/OWASP/joomscan

    如下图，Joomla的版本是3.9.12，同时扫描出后台路径：http://192.168.1.110/administrator，以及备份的配置文件http://192.168.1.110/configuration.php~。

    访问备份的配置文件发现了一个用户名和密码，应该是连接数据库用的。

    于是一条攻击思路就形成了：由于目标主机的3306端口是开放的，所以我们可以连接到数据库；再根据 joomla 官方文档中恢复或重置管理员密码的操作（如下图），添加一个网站管理员账号；最后登录后台Getshell。

    四、横向移动

    1、内网扫描

    这里使用msf生成一个elf，上传到服务器并赋予执行权限后执行，反弹接收一下。

    从前面Ubuntu的IP地址和这个CentOS的双网卡IP地址我们知道内网的网段是192.168.93.0/24，于是准备扫一下内网里的存活主机。

    添加路由后放后台，用smb扫一下内网

    发现了三台加入了TEST域的Windows主机，分别是：

    Windows 2012：192.168.93.10

    Windows 2008：192.168.93.20

    Windows 7 ：192.168.93.30

    我们就可以利用wmiexec登录到Windows2008经行域渗透了。

    2、内网信息收集

    利用msf开个sosk4a正向代理，配合proxychains利用wmiexec登录到Windows2008并执行命令。

    ipconfig /all看到DNS Servers是Windows2012，ping test.org看到域控IP也是Windows2012的，那么Windows2012应该就是域控。

    tasklist /V查看进程和所属用户，可以看到TEST域administrator的进程，就可以利用mimikatz抓一下密码。

    net share查看共享目录
